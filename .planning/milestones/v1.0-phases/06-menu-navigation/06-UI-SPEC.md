## Design System

| Property | Value |
|----------|-------|
| Tool | none — native Android MMD (Mudita Mindful Design) library |
| Preset | not applicable |
| Component library | MMD 1.0.1 — ButtonMMD, TextMMD, ThemeMMD |
| Icon library | none — text-only buttons throughout |
| Font | MMD default (E-ink-tuned — do not override) |

Source: CLAUDE.md §MMD Library, CONTEXT.md canonical_refs §Project constraints.

No shadcn gate applies. This is a Kotlin/Compose/Android project — the MMD library is the design system. All components must be ButtonMMD and TextMMD wrapped in ThemeMMD. No web component libraries are applicable.

## Typography

MMD manages actual font metrics. Phase code must use only `FontWeight.Bold` or `FontWeight.Normal` — no raw sp/px sizes.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Screen heading | MMD default | Bold (FontWeight.Bold) | MMD default |
| Button label | MMD default | Normal (FontWeight.Normal) | MMD default |
| Body / stat label | MMD default | Normal (FontWeight.Normal) | MMD default |
| Stat value (score) | MMD default | Bold (FontWeight.Bold) | MMD default — score rows only |

Rule: Do not pass explicit `fontSize` to TextMMD. MMD typography is tuned for E-ink readability — overriding sizes below MMD defaults violates CLAUDE.md §MMD Library.

Two weights only: `FontWeight.Bold` (headings, score value row) and `FontWeight.Normal` (all other text).

Source: SummaryScreen.kt lines 53–58 (heading Bold), LeaderboardScreen.kt lines 49–56 (heading Bold), ScoreRow body (Normal). CLAUDE.md: "do not override font sizes below MMD defaults."

## Color

E-ink display. Monochromatic only. No color values outside White/Black are permitted.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Color.White | Screen background; Surface fill for all cards and panels |
| Secondary (30%) | Color.White + Color.Black border (1dp) | Stats/score panel (Surface with BorderStroke) |
| Accent (10%) | none | No accent color — E-ink is monochromatic |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for: nothing — monochromatic E-ink. The eInkColorScheme provides no color. Never introduce a tinted accent; ThemeMMD will not render it meaningfully.

Selected state (MenuScreen conditional Resume button): presence/absence, not color change. "Resume" is simply absent from the layout when `hasSavedGame()` is false. No greyed-out or disabled color state is needed.

Source: CLAUDE.md §MMD Library — "eInkColorScheme (monochromatic) — do not use dynamicColorScheme or any color-tinted scheme." SummaryScreen.kt `Surface(color = Color.White, border = BorderStroke(1.dp, Color.Black))`.

## Screen Specifications

### MenuScreen

File: `app/src/main/java/com/mudita/sudoku/ui/game/MenuScreen.kt`

Layout: Full-screen Column with `windowInsetsPadding(WindowInsets.systemBars)` + `padding(horizontal = 16.dp)`.

| Element | Spec |
|---------|------|
| App title | TextMMD, text = "Sudoku", FontWeight.Bold, TextAlign.Center, fillMaxWidth, padding(top = 32.dp, bottom = 24.dp) |
| "New Game" button | ButtonMMD, fillMaxWidth, height = 56.dp — always visible |
| "Resume" button | ButtonMMD, fillMaxWidth, height = 56.dp — visible ONLY when `hasSavedGame()` is true; absent (not gone/greyed) otherwise |
| "Best Scores" button | ButtonMMD, fillMaxWidth, height = 56.dp — always visible |
| Button order | New Game → Resume (conditional) → Best Scores (top to bottom) |
| Spacer between buttons | Spacer(height = 8.dp) after each button |
| Vertical push | Spacer(weight = 1f) above the button group so buttons sit in the lower half, or title at top and buttons below with natural flow — Claude's discretion on exact weight placement, but buttons must not float in the middle of a blank screen |

Source: CONTEXT.md D-01, D-02, D-03.

## Interaction Contracts

### Back-Press Behavior

| Screen | Back-press result |
|--------|------------------|
| MENU | Exits the app (ComponentActivity default — no override needed) |
| DIFFICULTY | Navigates to MENU |
| GAME (mid-play) | Auto-saves via `viewModel.saveNow()` then navigates to MENU — no dialog |
| SUMMARY | Navigates to MENU |
| LEADERBOARD | Navigates to MENU |

Source: CONTEXT.md D-10, D-11, D-12.

### Conditional Resume Button

- `Resume` button renders only when `viewModel.hasSavedGame()` returns `true`.
- It is absent from the layout entirely (not visible-but-disabled) when false.
- Implementation: `if (hasSavedGame) { ButtonMMD(...) { TextMMD("Resume") } }` — no `alpha`, no `enabled = false`.

Source: CONTEXT.md D-02, specifics section.

### Navigation Flow

```
MENU --[New Game]--> DIFFICULTY --[Easy|Medium|Hard]--> GAME --[complete]--> SUMMARY
MENU --[Resume]----> GAME
MENU --[Best Scores]--> LEADERBOARD
SUMMARY --[View Leaderboard]--> LEADERBOARD
SUMMARY --[Back to Menu]--> MENU
LEADERBOARD --[Back to Menu]--> MENU
GAME --[back press]--> MENU (auto-save)
DIFFICULTY --[Back | back press]--> MENU
LEADERBOARD --[back press]--> MENU
SUMMARY --[back press]--> MENU
```

Source: CONTEXT.md D-08, D-09.

### Touch Targets

All interactive elements: minimum 56dp height, `fillMaxWidth`. This is a standing constraint (UI-03) established in Phase 3 and applied in all prior screens.

## E-Ink Compliance Checklist

These rules apply to every composable in this phase. They are not new — they are standing constraints from CLAUDE.md and prior phases.

| Rule | Requirement |
|------|-------------|
| No ripple | ButtonMMD suppresses it via ThemeMMD — do not override `indication` |
| No animations | No `AnimatedVisibility`, no `animate*AsState`, no `Crossfade`, no `Transition` |
| No color schemes | `eInkColorScheme` only via ThemeMMD — no `dynamicColorScheme`, no custom MaterialTheme |
| No spinners | LoadingScreen uses static text only (already implemented) |
| No transitions | `when(currentScreen)` switches instantly — no slide/fade between screens |
| Monochromatic only | Color.White and Color.Black only for any explicit color values |

Source: CLAUDE.md §MMD Library §Constraints, REQUIREMENTS.md UI-02.
