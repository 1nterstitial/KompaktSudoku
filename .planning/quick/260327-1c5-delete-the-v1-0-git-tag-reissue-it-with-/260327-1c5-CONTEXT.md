# Quick Task 260327-1c5: v1.0 Release Reissue + Signed APK - Context

**Gathered:** 2026-03-27
**Status:** Ready for planning

<domain>
## Task Boundary

Delete the v1.0 git tag, reissue it with a concise description, create a GitHub release, build a signed APK and attach it to the release.

</domain>

<decisions>
## Implementation Decisions

### APK Signing
- Generate a new keystore stored outside the repo (e.g. ~/kompakt-sudoku.jks)
- Keystore DN must contain NO personal or authorship info — use anonymous values (CN=KompaktSudoku, OU=Unknown, O=Unknown, L=Unknown, ST=Unknown, C=Unknown)
- Configure signing in app/build.gradle.kts via signingConfigs block
- Signing credentials (storePassword, keyPassword) sourced from local.properties or environment — never committed to repo

### Release Description Format
- One-liner only — concise, no bullet lists
- Must NOT include author/personal info
- Example style: "Fully playable Sudoku for Mudita Kompakt — 3 difficulty levels, pause/resume, leaderboard."

### Tag Description
- Delete local and remote v1.0 tag, recreate with concise one-liner message matching the GitHub release description

</decisions>

<specifics>
## Specific Requirements

- Keystore file location: ~/kompakt-sudoku.jks (outside repo, not committed)
- signing config in build.gradle.kts reads from local.properties (storeFile, storePassword, keyAlias, keyPassword)
- local.properties is already gitignored
- Build target: assembleRelease
- APK attached to GitHub release as an asset

</specifics>
