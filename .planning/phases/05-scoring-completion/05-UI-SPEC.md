# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Scoring & Completion.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Android/Compose — shadcn not applicable) |
| Preset | not applicable |
| Component library | MMD (Mudita Mindful Design) 1.0.1 — mandatory |
| Icon library | none — text labels only (E-ink display) |
| Font | MMD default (E-ink tuned — do not override) |

**Source:** CLAUDE.md §MMD Library (Mandatory), codebase inspection.

All UI components must be wrapped in ThemeMMD. Use ButtonMMD and TextMMD throughout.
No standard Material Button or Text. No animations. No ripple effects.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Inline label gaps |
| sm | 8dp | ControlsRow button gaps, Spacer between sections (established in GameScreen) |
| md | 16dp | Dialog vertical padding, section spacing within summary/leaderboard screens |
| lg | 24dp | Dialog horizontal padding (established in ResumeDialog) |
| xl | 32dp | Major screen-level vertical breathing room on summary/leaderboard screens |
| 2xl | 48dp | not used in this phase |
| 3xl | 64dp | not used in this phase |

Exceptions:
- NumberPad uses 2dp between digit buttons (10 buttons across 800px — tighter packing required to fit row)
- All interactive elements enforce 56dp minimum height regardless of spacing scale (UI-03 touch target floor)

**Source:** codebase — ControlsRow.kt (`Arrangement.spacedBy(8.dp)`), GameScreen.kt (`Spacer(height = 8.dp)`, `padding(horizontal = 8.dp)`), ResumeDialog (`padding(vertical = 16.dp, horizontal = 24.dp)`), NumberPad.kt (`Arrangement.spacedBy(2.dp)`).

## Typography

| Role | Size | Weight | Line Height | Color |
|------|------|--------|-------------|-------|
| Grid digit — given cell | 20sp | Bold (700) | single line | Black (unselected) / White (selected) |
| Grid digit — player fill | 20sp | Normal (400) | single line | Black (unselected) / White (selected) |
| Pencil mark candidate | 9sp | Normal (400) | single line | Black only |
| UI label (buttons, dialog text, screen headings) | MMD default | MMD default | MMD default | Black |
| Score value (large number on summary screen) | 20sp | Bold (700) | single line | Black |
| Section heading (summary screen) | MMD default | Bold (700) | MMD default | Black |
| Stat row label + value (summary, leaderboard) | MMD default | Normal (400) | MMD default | Black |

Notes:
- MMD default typography is E-ink tuned — do not override font sizes below MMD defaults.
- No custom font families. No font size overrides on ButtonMMD or TextMMD label text.
- Bold weight is used only for: given cell digits, final score value, and screen headings.

**Source:** GameGrid.kt (`fontSize = 20.sp`, `FontWeight.Bold` for given, `FontWeight.Normal` for player, `fontSize = 9.sp` for pencil marks), CLAUDE.md §MMD Library.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | White (`Color.White`) | Screen background, cell default fill, button default surface |
| Secondary (30%) | Black (`Color.Black`) | Grid lines, border strokes, text, button outlines |
| Accent (10%) | Black (`Color.Black`) — inverted fill | Selected cell background + active mode toggle background only |
| Destructive | not applicable | No destructive actions in Phase 5 |

Accent reserved for:
1. Selected cell in GameGrid (solid black cell fill, white digit text)
2. Active mode button in ControlsRow (solid black fill, white label text)
3. No other elements

**E-ink constraint:** The Mudita Kompakt display is monochromatic. There is no color palette.
All visual differentiation is achieved through fill inversion (black/white swap) and stroke weight
(1dp thin cell borders, 2.5dp thick box borders, 1dp inset error indicator).

New screen surfaces (SummaryScreen, LeaderboardScreen):
- Background: Color.White (dominant)
- Card/panel surface: Color.White with BorderStroke(1.dp, Color.Black) — matches ResumeDialog pattern
- All text: Color.Black

**Source:** GameGrid.kt (color usage), ControlsRow.kt (active/inactive fill), GameScreen.kt (ResumeDialog: `Surface(color = Color.White, border = BorderStroke(1.dp, Color.Black))`), CLAUDE.md §MMD.

## Screen Contracts

### Hint Button (ControlsRow extension)

**Placement:** Fourth button in ControlsRow — `[ Fill ] [ Pencil ] [ Undo ] [ Hint ]`
**Component:** ButtonMMD (same as Undo — stateless, no active/inactive color inversion)
**Modifier:** `Modifier.weight(1f).sizeIn(minHeight = 56.dp)` — identical to Undo button
**Label:** "Hint"
**Disabled state:** ButtonMMD disabled (greyed by MMD internals) when no valid hint target exists
**Source:** 05-CONTEXT.md D-02

### SummaryScreen

**Layout:** Full-screen Column with `windowInsetsPadding(WindowInsets.systemBars)` + `padding(horizontal = 16.dp)`

**Content order (top to bottom):**
1. Screen heading: "Puzzle Complete" — TextMMD, Bold, centered, `padding(top = 32.dp, bottom = 24.dp)`
2. Difficulty label: difficulty name (e.g., "Easy") — TextMMD, centered
3. Stats panel — Surface with `BorderStroke(1.dp, Color.Black)`, `padding(16.dp)`:
   - Row: "Errors" label + errorCount value
   - Row: "Hints" label + hintCount value
   - Divider: 1dp Black horizontal line
   - Row: "Score" label + finalScore value (finalScore displayed Bold)
4. Personal best notice (conditional): TextMMD "New personal best!" — shown only when `isPersonalBest == true`, centered, `padding(vertical = 16.dp)`
5. Spacer — `weight(1f)` to push buttons to bottom
6. Action buttons (full width, 56dp height, 8dp gap between):
   - "View Leaderboard" — ButtonMMD
   - "New Game" — ButtonMMD

**Source:** 05-CONTEXT.md D-06

### LeaderboardScreen

**Layout:** Full-screen Column with `windowInsetsPadding(WindowInsets.systemBars)` + `padding(horizontal = 16.dp)`

**Content order (top to bottom):**
1. Screen heading: "Best Scores" — TextMMD, Bold, centered, `padding(top = 32.dp, bottom = 24.dp)`
2. Scores panel — Surface with `BorderStroke(1.dp, Color.Black)`, `padding(16.dp)`:
   - Row: "Easy" label + score value (or "—" if no score recorded)
   - 1dp Black divider
   - Row: "Medium" label + score value (or "—")
   - 1dp Black divider
   - Row: "Hard" label + score value (or "—")
3. Spacer — `weight(1f)` to push button to bottom
4. Action button (full width, 56dp height):
   - "New Game" — ButtonMMD

**Source:** 05-CONTEXT.md D-07

### Score Row Pattern (reused in both screens)

```
Row(
    modifier = Modifier.fillMaxWidth().padding(vertical = 8.dp),
    horizontalArrangement = Arrangement.SpaceBetween
) {
    TextMMD(text = label)
    TextMMD(text = value)
}
```

All stat rows use this pattern — label left-aligned, value right-aligned.

## Interaction Contracts

### No-ripple rule (UI-02)
All clickable elements use `indication = null` or ButtonMMD (which suppresses ripple via ThemeMMD).
No `Animated*` composable anywhere. No `AnimatedVisibility`. No `animateContentSize`.

### Touch target floor (UI-03)
Every tappable element: minimum 56dp height.
- Hint button: `sizeIn(minHeight = 56.dp)` via Modifier (matches Undo pattern)
- Summary/Leaderboard buttons: `Modifier.fillMaxWidth().height(56.dp)` (matches ResumeDialog button pattern)

### Screen routing (MainActivity)
A top-level `Screen` enum (`GAME`, `SUMMARY`, `LEADERBOARD`) drives which composable is active.
- `GAME → SUMMARY`: triggered when `GameEvent.Completed` is collected in LaunchedEffect
- `SUMMARY → LEADERBOARD`: "View Leaderboard" button callback
- `LEADERBOARD → GAME`: "New Game" button callback
- `SUMMARY → GAME` (direct): "New Game" button callback on summary screen

Phase 6 replaces the Screen enum with NavHost. Leaf composables (SummaryScreen, LeaderboardScreen) must not contain nav-awareness — they receive data as parameters and emit callbacks.

**Source:** 05-CONTEXT.md D-05.

### Hint button disabled state
When all unfilled cells already contain the correct value (no valid hint target), the Hint button is
disabled. Use `ButtonMMD(enabled = canRequestHint)`. MMD renders disabled state — do not implement
custom disabled appearance.
