# Phase 8 — UI Design Contract

> Visual and interaction contract for Phase 8: Controls & Number Pad Fixes.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Android/Compose project; shadcn is not applicable |
| Preset | not applicable |
| Component library | Mudita Mindful Design (MMD) 1.0.1 — ButtonMMD, TextMMD |
| Icon library | none — text labels only |
| Font | system default via MMD; `sans-serif-condensed` (FontFamily) for NumberPad digit buttons |

**E-ink constraints (mandatory — source: CLAUDE.md):**
- No ripple effects — all `clickable` calls use `indication = null`
- No animations — no `Animated*` composables
- No `RoundedCornerShape` — use `RectangleShape` exclusively to avoid anti-aliased gray pixels
- No color tinting — monochromatic only (`Color.Black`, `Color.White`, `Color(0xFFE0E0E0)`)
- `ButtonMMD` and `TextMMD` preferred; fall back to `Box + clickable(indication=null)` when MMD API is unverifiable at compile time (established pattern from ControlsRow.kt)

## Typography

| Role | Size | Weight | Line Height | Font Family | Usage |
|------|------|--------|-------------|-------------|-------|
| Button label | MMD default (do not override) | MMD default | MMD default | MMD default | ControlsRow: Undo, Get Hint labels via TextMMD |
| Mode toggle label | MMD default | MMD default | MMD default | MMD default | ControlsRow: "Fill", "Pencil" labels via TextMMD |
| Number pad digit | MMD default size | MMD default weight | MMD default | `sans-serif-condensed` | NumberPad digits 1–9 and × erase button |
| Hint button two-line | MMD default | MMD default | MMD default | MMD default | "Get\nHint" label — two lines, centered alignment |

**TextMMD font override risk (source: STATE.md known risks):**
- TextMMD closed-source AAR may silently ignore `fontSize`, `lineHeight`, or `fontFamily` overrides.
- Attempt `TextMMD(text = "1", fontFamily = FontFamily(Typeface.create("sans-serif-condensed", Typeface.NORMAL)))` first for CTRL-01.
- If the condensed font is ignored (verified by visual inspection on device or emulator), fall back to `BasicText` inside a centered `Box` — same `indication = null` pattern established in ControlsRow.kt for mode toggles.
- Document the result as a CLAUDE.md convention after Phase 8 completion.

**"Get Hint" two-line label (CTRL-02):**
- Label string: `"Get\nHint"` — newline character produces two lines.
- `textAlign = TextAlign.Center` so both lines center within the button.
- If TextMMD does not expose `textAlign`, wrap in a `Column(horizontalAlignment = Alignment.CenterHorizontally)` with two separate `TextMMD` calls: `TextMMD("Get")` and `TextMMD("Hint")`.

| State | Background | Text color |
|-------|-----------|------------|

Current code uses `Color.White` for inactive background — this is the bug CTRL-03 fixes. Change inactive from `Color.White` to `Color(0xFFE0E0E0)`.

## Component Inventory

### ControlsRow.kt changes

**CTRL-03: Inactive mode button background**
- Change: `Color.White` → `Color(0xFFE0E0E0)` for the inactive mode toggle box background.
- Applies to both the Fill box and the Pencil box (whichever is not the current `inputMode`).
- No change to the active state (remains `Color.Black` background, `Color.White` text).

**CTRL-04: Fill+Pencil group border frame**
- Wrap the Fill `Box` and Pencil `Box` in a shared `Box` (or `Row`) with `Modifier.border(width = 1.dp, color = Color.Black, shape = RectangleShape)`.
- The border frame encloses ONLY the Fill and Pencil boxes — not Undo, not Get Hint.
- Use `RectangleShape` — no rounded corners.
- Gap between the Fill+Pencil group and the Undo/Get Hint buttons: existing `Arrangement.spacedBy(8.dp)` is sufficient.

### NumberPad.kt changes

**CTRL-01: Digit vertical centering + condensed font**
- Replace `TextMMD(text = digit.toString())` with a version that applies `fontFamily = FontFamily(Typeface.create("sans-serif-condensed", Typeface.NORMAL))`.
- Vertical centering: ButtonMMD's content lambda should already center via `contentAlignment = Alignment.Center` — if digits appear off-center, add explicit `Modifier.fillMaxHeight()` on the content or switch to `Box(contentAlignment = Alignment.Center)` wrapping the TextMMD.
- If TextMMD ignores the fontFamily override, fall back to `BasicText` with a `TextStyle` that includes `fontFamily` and `textAlign = TextAlign.Center`.

**CTRL-02: Get Hint two-line label**
- This change is in `ControlsRow.kt`, not `NumberPad.kt`.
- Current: `TextMMD(text = "Get Hint")` — single line.
- Required: Two centered lines.
- Preferred: `TextMMD(text = "Get\nHint", textAlign = TextAlign.Center)` if TextMMD exposes `textAlign`.
- Fallback: `Column(horizontalAlignment = Alignment.CenterHorizontally) { TextMMD("Get"); TextMMD("Hint") }`.

## Interaction Contract

**All buttons in ControlsRow and NumberPad:**
- `indication = null` — no ripple, no touch feedback animation (E-ink ghosting prevention)
- `interactionSource = remember { MutableInteractionSource() }` — required when `indication = null`
- Minimum touch target: 56dp height via `sizeIn(minHeight = 56.dp)` — must be preserved on all modified buttons

**Fill/Pencil mode toggles:**
- Tap on active mode button: no-op (already active — existing behavior preserved)
- Tap on inactive mode button: calls `onToggleMode()` — existing behavior preserved
- Visual state change is immediate on recomposition — no transition animation

**Get Hint button:**
- Disabled when `canRequestHint == false` — existing behavior preserved
- Disabled state appearance: MMD handles via `ButtonMMD(enabled = false)` — do not add custom disabled styling

**NumberPad:**
- All 10 buttons (1–9 + ×) always enabled — no disabled state needed
- Weight(1f) distribution preserved — each button gets equal width
