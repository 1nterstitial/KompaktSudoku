# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Game State & Domain.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Phase scope note:** Phase 2 is a pure ViewModel and domain model layer —
> no Compose composables are authored here. This spec defines the state-to-visual
> mappings, interaction semantics, copywriting, and touch target contracts that
> the Phase 3 UI must honour. The executor implementing Phase 2 must embed these
> contracts in `GameUiState` field names, sealed class variants, and ViewModel
> action signatures so Phase 3 has zero ambiguity about what to render.

## Spacing Scale

Declared values (multiples of 4, in dp — this is an Android/Compose project):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gaps, inline element spacing |
| sm | 8dp | Compact element spacing within a cell group |
| md | 16dp | Default padding between UI sections |
| lg | 24dp | Section padding (number pad to grid gap) |
| xl | 32dp | Layout-level gaps |
| 2xl | 48dp | Major section breaks |
| 3xl | 64dp | Page-level vertical spacing |

Exceptions:
- Cell touch target minimum: 56dp square (REQUIREMENTS UI-03 — mandated, not optional)
- Number pad button touch target minimum: 56dp square (REQUIREMENTS UI-03)
- Mode toggle touch target minimum: 56dp height (REQUIREMENTS UI-03)
- Undo button touch target minimum: 56dp square (REQUIREMENTS UI-03)

All touch targets at 56dp are non-negotiable per UI-03. The 800x480 E-ink display
requires reliable single-tap activation without hover feedback.

## Typography

MMD's ThemeMMD provides the type scale. Do not override font sizes below MMD defaults.
The values below are the role-to-MMD-style mappings for this phase's state labels.

| Role | MMD Style | Weight | Line Height | Usage |
|------|-----------|--------|-------------|-------|
| Cell digit | MMD body / headline equivalent | semibold (600) | 1.0 (single line) | Filled cell value, given cell value |
| Pencil mark digit | MMD label / caption equivalent | regular (400) | 1.0 | Candidate digits inside a cell (smaller) |
| Mode label | MMD label | regular (400) | 1.2 | "Fill" / "Pencil" mode toggle label |
| Action label | MMD label | semibold (600) | 1.2 | "Undo" button label |

Rules:
- Exactly 2 weights in use: regular (400) and semibold (600).
- Cell digit is larger than pencil mark digits — MMD's size differential must make
  the two visually distinct without color (monochromatic).
- Do not introduce a third weight.

## Color

The display is monochromatic E-ink. All color decisions reduce to black/white/grey
contrast only. ThemeMMD's eInkColorScheme governs this entirely.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | White (#FFFFFF via eInkColorScheme) | Cell backgrounds, screen background |
| Secondary (30%) | Light grey (eInkColorScheme surface variant) | Given cell backgrounds, number pad backgrounds |
| Accent (10%) | Black (#000000 via eInkColorScheme primary) | Selected cell highlight border, active mode indicator |
| Destructive | Not applicable in Phase 2 | No destructive actions in game loop (errors are silent per SCORE-01) |

Accent reserved for:
1. Selected cell border/highlight (INPUT-01 — selected cell must be visually highlighted)
2. Active input mode indicator (INPUT-03 — user must know which mode is active)

Never use accent for:
- Given cells (they use secondary background, not accent)
- Error indication during play (errors are silent — SCORE-01, REQUIREMENTS out-of-scope table)
- Pencil mark text (use same foreground as body, just smaller)

## State-to-Visual Mapping Contract

This section defines what each `GameUiState` field drives visually.
Phase 3 must implement exactly these mappings. Phase 2 must expose exactly these fields.

### Cell Visual States

A cell must be renderable in exactly 5 mutually exclusive visual states:

| State | `GameUiState` condition | Visual |
|-------|------------------------|--------|
| Given | `givenMask[index] == true` | Secondary background, semibold digit, not tappable for input |
| Empty unselected | `board[index] == 0 && selectedCellIndex != index` | White background, no digit |
| Empty selected | `board[index] == 0 && selectedCellIndex == index` | Accent border (high contrast black border), no digit |
| Filled unselected | `board[index] != 0 && !givenMask[index] && selectedCellIndex != index` | White background, semibold digit |
| Filled selected | `board[index] != 0 && !givenMask[index] && selectedCellIndex == index` | Accent border, semibold digit |

Pencil mark cells overlay the above: when `pencilMarks[index].isNotEmpty()` and
`board[index] == 0`, render pencil mark digits (regular weight, smaller) inside the cell
in a 3x3 mini grid layout instead of a single digit. The selected accent border still
applies when `selectedCellIndex == index`.

### Input Mode Visual State

The mode toggle must show exactly one of two states at all times:

| `inputMode` value | Label | Visual treatment |
|-------------------|-------|-----------------|
| `InputMode.FILL` | "Fill" | Accent border or filled background on "Fill" option |
| `InputMode.PENCIL` | "Pencil" | Accent border or filled background on "Pencil" option |

The inactive mode option uses secondary background. The active mode option uses accent treatment.
Never show both as active or neither as active.

### Loading State

When `isLoading == true` (puzzle generation in progress):
- Display a non-animated static loading indicator: "Generating puzzle..." text only
- Disable all cell tap targets and number pad buttons
- Do not use a spinner or progress animation (E-ink — UI-02)

When `isLoading == false`: render the board normally.

### Completion State

When `isComplete == true`:
- `GameEvent.Completed` is emitted via SharedFlow
- Phase 3 navigation handles the transition to summary screen
- No in-game celebration animation or overlay (UI-02 — no animations)

## Interaction Semantics

### Cell Tap (INPUT-01, INPUT-02)

| Condition | Action | Result |
|-----------|--------|--------|
| Tap given cell | `selectCell(index)` — but given cells are not selectable for input | No selection change; given cells reject digit input |
| Tap empty/filled non-given cell | `selectCell(index)` | `selectedCellIndex` updates immediately; previous selection clears |
| Tap already-selected cell | `selectCell(index)` | Selection remains on the same cell (no deselect on re-tap) |
| Tap outside grid | Not modeled in Phase 2 | Phase 3 may deselect on outside tap — leave `selectedCellIndex = null` |

Given cells must not accept digit input. The ViewModel `enterDigit()` must guard:
`if (state.givenMask[state.selectedCellIndex!!]) return`.

### Digit Entry (INPUT-02, INPUT-04)

| Condition | Action |
|-----------|--------|
| `inputMode == FILL`, cell selected, cell not given | Fills `board[selectedCellIndex]`; pushes `GameAction.FillCell` to undo stack; checks error (SCORE-01) and completion (SCORE-02) |
| `inputMode == PENCIL`, cell selected, cell not given | Toggles digit in `pencilMarks[selectedCellIndex]`; pushes `GameAction.PencilToggle` to undo stack |
| No cell selected | Digit tap is a no-op |
| Cell is given | Digit tap is a no-op |
| `board[selectedCellIndex] != 0` and mode is FILL | Overwrites existing fill; old value is saved in `GameAction.FillCell` for undo |

### Undo (INPUT-05)

Undo reverses the last action on `undoStack`. The undo button:
- Is enabled when `undoStack.isNotEmpty()`
- Is disabled (visually and functionally) when `undoStack.isEmpty()`
- Label: "Undo" (no icon required; text label sufficient per REQUIREMENTS)

### Mode Toggle (INPUT-03)

Tapping the mode toggle calls `toggleInputMode()`.
- Switches from FILL to PENCIL or PENCIL to FILL.
- No confirmation required.
- The mode toggle is always enabled (never disabled during active play).

### Error Tracking (SCORE-01)

Errors are tracked silently. No visual feedback is given to the player when an incorrect
digit is entered. The `errorCount` field in `GameUiState` is NOT rendered anywhere in
the play UI. It is surfaced only at completion (Phase 5 scope). The executor must not
render `errorCount` in any Phase 2 or Phase 3 screen component during active play.
