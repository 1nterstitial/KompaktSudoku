  <action>
    Present the physical device verification checklist to the user. The complete game UI has been built and verified via automated tests (Plans 01-04). The following E-ink-specific behaviors CANNOT be verified by automated tests and require physical Mudita Kompakt hardware (or a compatible E-ink emulator).

    **Prerequisites before testing:**
    - MMD library must be resolved (either via GitHub token or local cache)
    - APK must be built: `./gradlew.bat :app:assembleDebug`
    - APK must be installed on physical Mudita Kompakt device

    If physical hardware is not available yet, user responds "deferred" to skip this checkpoint and revisit when hardware is available.

    **Checklist to present to user:**

    **A. Ghosting / Display Artifacts (Phase 3 Success Criterion 4)**

    1. Start a new game (any difficulty)
    2. Tap 30+ different cells in sequence (move selection around the grid rapidly)
    3. After 30+ taps, pause and inspect the display:
       - [ ] No ghost images of previously selected cells remain visible
       - [ ] The currently selected cell shows a clean solid black fill
       - [ ] Previously selected cells show no residual darkening or shadow
    4. Enter digits in 10+ cells, then use Undo repeatedly:
       - [ ] No ghost digits remain visible after undo clears them
    5. Toggle between Fill and Pencil modes 10+ times:
       - [ ] Mode toggle buttons show clean state transitions (no blending of active/inactive states)

    **B. Animation / Ripple Absence (Phase 3 Success Criterion 2)**

    6. Tap any digit button (1-9) on the number pad:
       - [ ] No ripple effect, color wave, or expanding circle appears on the button
       - [ ] Button press feedback is instant (no fade-in or fade-out transition)
    7. Tap the Erase button:
       - [ ] No ripple or animation on press
    8. Tap Fill and Pencil mode toggle buttons:
       - [ ] State change is instantaneous (no crossfade or animated color transition)
    9. Tap the Undo button:
       - [ ] No ripple or animation on press
    10. Tap grid cells to select them:
        - [ ] Selection highlight appears instantly (no animated transition from previous to new selection)
    11. Start a new game and observe the loading state:
        - [ ] "Generating puzzle..." text appears statically (no spinner, no pulsing, no dots animation)

    **C. Touch Target Reliability (Phase 3 Success Criterion 3)**

    12. Tap each digit button (1-9) with a single deliberate tap:
        - [ ] Each button registers on first tap without requiring double-tap or long-press
    13. Tap the Erase button with a single tap:
        - [ ] Registers reliably on first tap
    14. Tap Fill/Pencil toggle with a single tap:
        - [ ] Registers reliably on first tap
    15. Tap the Undo button with a single tap:
        - [ ] Registers reliably on first tap
    16. Tap grid cells at the corners (top-left, top-right, bottom-left, bottom-right):
        - [ ] Each corner cell registers on first tap
    17. Tap grid cells at the center:
        - [ ] Center cells register on first tap
    18. Tap cells along the thick 3x3 box borders:
        - [ ] Cells adjacent to thick borders register correctly (tap is not absorbed by the border)

    **D. Visual Rendering Quality**

    19. Inspect the grid border rendering:
        - [ ] 3x3 box borders are visibly thicker than individual cell borders
        - [ ] Borders appear crisp (not blurry or feathered) on E-ink display
    20. Inspect digit rendering:
        - [ ] Given (pre-filled) digits appear in bold weight
        - [ ] Player-entered digits appear in regular weight
        - [ ] Selected cell shows white digit on black background with good contrast
    21. Inspect pencil marks:
        - [ ] Pencil marks are readable (not too small) on the E-ink display
        - [ ] 3x3 mini-grid layout within cells is distinguishable
    22. Inspect the number pad layout:
        - [ ] All 10 buttons (1-9 + Erase) are visible and fit within the screen width
        - [ ] Button labels are readable on E-ink
    23. Check ThemeMMD rendering:
        - [ ] Color scheme is monochromatic (black, white, grays only -- no color tints)
        - [ ] Typography is crisp and readable at E-ink resolution

    **E. Layout on 800x480 Display**

    24. Verify overall layout fits:
        - [ ] Difficulty label visible at top
        - [ ] Grid is the largest element and fills available vertical space
        - [ ] Controls row (Fill/Pencil/Undo) is visible below grid
        - [ ] Number pad is visible at bottom
        - [ ] No content is clipped by navigation bar or system bars
        - [ ] Bottom padding accounts for Mudita navigation bar (~20dp)

    **User response options:**
    - "approved" — all checklist items pass on physical hardware
    - "issues: [describe specific failures]" — some items fail, describe which ones
    - "deferred" — physical hardware not available yet; skip and revisit later
